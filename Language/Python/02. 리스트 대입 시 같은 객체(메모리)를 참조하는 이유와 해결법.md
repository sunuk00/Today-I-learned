```python
import this

x = ['orange', 'banana', 'apple']
y = x
y.append('kill')

print(f'x == y : { x == y}') # True
print(f'x is y : {x is y}') # True # is 는 같은 object를 가리키고 있는지를 확인

print(f'x value, id : {x}, {hex(id(x))}') # 가리키는 객체가 같음..
print(f'y value, id : {y}, {hex(id(y))}')
```

출력:
```
x == y : True
x is y : True
x value, id : ['orange', 'banana', 'apple', 'kill'], 0x2517e1b2dc0
y value, id : ['orange', 'banana', 'apple', 'kill'], 0x2517e1b2dc0
```

출력을 보면 알겠지만, y에 `append('kill')`을 해주면 x에도 `kill`이 추가 되고, 둘의 id, 즉 참조하는 메모리가 같은 것을 확인할 수 있다.    
즉, 파이썬에서 리스트를 단순히 `=` 기호로 대입하면, 값이 복사되는 것이 아니라 **동일한 객체에 대한 참조(메모리 주소)가 복사**된다.

### 해결 with slicing and `.copy()`
이를 해결하기 위해서는 슬라이싱 혹은 `.copy()`를 사용하면 된다.

```python
# .copy()
x = ['orange', 'banana', 'apple']
y = x.copy()

print(f'x == y : { x == y}') # True
print(f'x is y : {x is y}') # False
y.append('kill')

print(f'x value, id : {x}, {hex(id(x))}') # 가리키는 객체가 다름
print(f'y value, id : {y}, {hex(id(y))}\n')


# slicing
x = ['orange', 'banana', 'apple']
y = x[:]

print(f'x == y : { x == y}') # True
print(f'x is y : {x is y}') # False
y.append('kill')

print(f'x value, id : {x}, {hex(id(x))}') # 가리키는 객체가 다름
print(f'y value, id : {y}, {hex(id(y))}')
```

출력결과:
```
x == y : True
x is y : False
x value, id : ['orange', 'banana', 'apple'], 0x2517d97c640
y value, id : ['orange', 'banana', 'apple', 'kill'], 0x2517e1e0980

x == y : True
x is y : False
x value, id : ['orange', 'banana', 'apple'], 0x2517e1f7a40
y value, id : ['orange', 'banana', 'apple', 'kill'], 0x2517e1cff80
```
